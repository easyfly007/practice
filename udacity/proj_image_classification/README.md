    <article class="markdown-body entry-content" itemprop="text"><h2><a id="user-content-running-the-udacity-deep-learning-foundations-image-classification-project-on-floydhubcom" class="anchor" href="#running-the-udacity-deep-learning-foundations-image-classification-project-on-floydhubcom" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Running the Udacity Deep Learning Foundations image classification project on floydhub.com</h2>
<ol>
<li>
<p>Create an account on <a href="https://www.floydhub.com">floydhub.com</a> (don't forget to confirm your email). You will automatically receive 100 free GPU hours.</p>
</li>
<li>
<p>Install the <code>floyd</code> command on your computer:</p>
<pre><code> pip install -U floyd-cli
</code></pre>
<p>Do this even if you already installed <code>floyd-cli</code> before, just to make sure you have the most recent version (Its pace of development is fast!).</p>
</li>
<li>
<p>Associate the command with your Floyd account:</p>
<pre><code> floyd login
</code></pre>
<p>(a page with authentication token will open; you will need to copy the token into your terminal)</p>
</li>
<li>
<p>Clone this repository:</p>
<pre><code> git clone https://github.com/ludwiktrammer/deep-learning.git
</code></pre>
<p>Note: There are couple minor differences between this repository and the original Udacity repository. You can read about them <a href="https://github.com/ludwiktrammer/deep-learning/tree/master/image-classification#how-is-this-repository-different-from-the-original">in README</a>. To follow this instructions you need to use this repository.</p>
</li>
<li>
<p>Enter the folder for the image classification project:</p>
<pre><code> cd image-classification
</code></pre>
</li>
<li>
<p>Initiate a Floyd project:</p>
<pre><code> floyd init dlnd_image_classification
</code></pre>
</li>
<li>
<p>Run the project:</p>
<pre><code> floyd run --gpu --env tensorflow --mode jupyter --data diSgciLH4WA7HpcHNasP9j
</code></pre>
<p>It will be run on a machine with GPU (<code>--gpu</code>), using a Tenserflow environment (<code>--env tensorflow</code>), as a Jupyter notebook (<code>--mode jupyter</code>), with Floyd's built-in cifar-10 dataset  available (<code>--data diSgciLH4WA7HpcHNasP9j</code>).</p>
</li>
<li>
<p>Wait for the Jupyter notebook to become available and then access the URL displayed in the terminal (described as "path to jupyter notebook"). You will see the notebook.</p>
</li>
<li>
<p>Remember to explicitly stop the experiment when you are not using the notebook. As long as it runs (even in the background) it will cost GPU hours. You can stop an experiment in the <a href="https://www.floydhub.com/experiments">"Experiments" section on floyd.com</a> or using the <code>floyd stop</code> command:</p>
<pre><code> floyd stop ID
</code></pre>
<p>(where ID is the "RUN ID" displayed in the terminal when you run the project; if you lost it you can also find it in the <a href="https://www.floydhub.com/experiments">"Experiments" section on floyd.com</a>)</p>
</li>
</ol>
<p><strong>Important:</strong> When you run a project it will always start from scratch (i.e. from the state present <em>locally</em> on your computer). If you made changes in the remote jupiter notebook during a previous run, the changes will <strong>not</strong> be present in subsequent runs. To make them permanent you need to add the changes to your local project folder. When running the notebook you can download them directly from Jupyter - <em>File / Download / Notebook</em>. After downloading it, just replace your local <code>dlnd_image_classification.ipynb</code> file with the newly downloaded one.</p>
<p>Alternatively, If you already stoped the experiment, you can still download the file using the <code>floyd output</code> command:</p>
